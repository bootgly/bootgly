# Documentation guidelines

## Reports

### Feasibility reports

Feasibility reports are intended to be a quick, high-level overview of the future features of a project. 

The report should contain an analysis of the development of a feature integrated with existing features...

The report should answer whether the feature to be developed is the next logical development step, whether the existing features already developed support the development of the feature, and how the feature would be structured (i.e., where it would be located) in the current Bootgly filesystem...

Structure of a feasibility report:

- Is <Feature> the Logical Next Step?
- Does the Existing Codebase Support This Feature?
- Proposed Filesystem Structure
- Component Design Overview
- Integrations
- Risks and Considerations
- Summary
